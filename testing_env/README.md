# Prolog Test Automation Scripts

## Introduction
This folder contains a set of scripts designed to compile, run, and clean up tests for comparing the current implementation against CLPQ. These scripts streamline the testing process, manage execution timeouts, and handle the generation and cleanup of test-related files.

## Contents


### Scripts

1. **`compile.sh <time>`**
Compiles all necessary files required for running the tests and sets the Prolog execution timeout.
**Parameters**:
    - `<time>`: The timeout value (in seconds) for Prolog execution. This value sets the maximum time allowed for each Prolog test run, helping to prevent tests from running indefinitely.

2. **`run_tests.sh <time>`**
   Executes a series of tests, comparing the current implementation against CLPQ until all tests are completed or a timeout occurs. Results are aggregated and saved in a `logs.txt` file
  
   **Parameters**:
     - `<time>`: The timeout value (in seconds) for individual test executions.

3. **`clean.sh`**
   Cleans up the working directory by removing all files generated by `./compile.sh`.

4. **`testing.pl`**
    The source code that use the structure of tests to compare the current implementation with CLPQ.

5. **`glpk-without-prolog.c`**
   C program that contains a specific test case where the GLPK solver takes an extended amount of time to compute results.

## Usage
- **Run the tests** using `./run_tests.sh <time>`. Review the results in `logs.txt`. This script already includes the compilation process. 
- **Compile the necessary files** with `./compile.sh <time>`, specifying the desired timeout value.
